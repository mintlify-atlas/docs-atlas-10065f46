## Prerequisites

- Java 11 or later
- Maven 3.x or Gradle 6+
- An existing JAX-RS project (Jersey, RESTEasy, or Apache CXF)

## Add the dependency

<Steps>
  <Step title="Add swagger-jaxrs2 to your build">
    <CodeGroup>
      ```xml Maven pom.xml
      <dependency>
        <groupId>io.swagger.core.v3</groupId>
        <artifactId>swagger-jaxrs2</artifactId>
        <version>2.2.46</version>
      </dependency>
      ```

      ```groovy Gradle build.gradle
      implementation 'io.swagger.core.v3:swagger-jaxrs2:2.2.46'
      ```
    </CodeGroup>

    <Tip>
      If your project uses Jakarta EE (jakarta.ws.rs), use the `-jakarta` artifact instead:
      `io.swagger.core.v3:swagger-jaxrs2-jakarta:2.2.46`
    </Tip>
  </Step>

  <Step title="Annotate a JAX-RS resource">
    Add `@Operation` and `@ApiResponse` to your resource methods:

    ```java PetResource.java
    import io.swagger.v3.oas.annotations.Operation;
    import io.swagger.v3.oas.annotations.media.Content;
    import io.swagger.v3.oas.annotations.media.Schema;
    import io.swagger.v3.oas.annotations.responses.ApiResponse;
    import io.swagger.v3.oas.annotations.tags.Tag;

    import javax.ws.rs.GET;
    import javax.ws.rs.Path;
    import javax.ws.rs.PathParam;
    import javax.ws.rs.Produces;
    import javax.ws.rs.core.MediaType;

    @Path("/pets")
    @Tag(name = "pets", description = "Pet management operations")
    public class PetResource {

        @GET
        @Path("/{id}")
        @Produces(MediaType.APPLICATION_JSON)
        @Operation(
            summary = "Find a pet by ID",
            description = "Returns a single pet by its identifier."
        )
        @ApiResponse(
            responseCode = "200",
            description = "Pet found",
            content = @Content(schema = @Schema(implementation = Pet.class))
        )
        @ApiResponse(responseCode = "404", description = "Pet not found")
        public Pet getPetById(@PathParam("id") long id) {
            // your implementation
            return petService.find(id);
        }
    }
    ```
  </Step>

  <Step title="Register OpenApiResource in your Application">
    Register `OpenApiResource` in your JAX-RS `Application` class so the spec is served at runtime:

    ```java MyApplication.java
    import io.swagger.v3.jaxrs2.integration.resources.OpenApiResource;

    import javax.ws.rs.ApplicationPath;
    import javax.ws.rs.core.Application;
    import java.util.HashSet;
    import java.util.Set;

    @ApplicationPath("/api")
    public class MyApplication extends Application {

        @Override
        public Set<Class<?>> getClasses() {
            Set<Class<?>> classes = new HashSet<>();
            // your resource classes
            classes.add(PetResource.class);
            // Swagger Core endpoint
            classes.add(OpenApiResource.class);
            return classes;
        }
    }
    ```
  </Step>

  <Step title="Fetch the generated spec">
    Start your application and retrieve the OpenAPI document:

    ```bash
    # JSON format
    curl http://localhost:8080/api/openapi.json

    # YAML format
    curl http://localhost:8080/api/openapi.yaml
    ```

    The response will be a fully formed OpenAPI 3.0 document describing all your annotated resources.
  </Step>
</Steps>

## Configure resource scanning

By default, Swagger Core scans all packages. To limit scanning to specific packages, configure `SwaggerConfiguration`:

```java
import io.swagger.v3.jaxrs2.integration.JaxrsOpenApiContextBuilder;
import io.swagger.v3.oas.integration.SwaggerConfiguration;

SwaggerConfiguration config = new SwaggerConfiguration()
    .resourcePackages(Set.of("com.example.resources"))
    .prettyPrint(true);

new JaxrsOpenApiContextBuilder<>()
    .openApiConfiguration(config)
    .buildContext(true);
```
